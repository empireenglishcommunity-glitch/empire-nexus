### SC1 — Zero non-Arabic-script output reaches a student

**Status: ✅ VERIFIED — passes unconditionally.**

Phase A4's 200-iteration stress test (re-confirmed passing in A8's own
full suite run, 717/717 tests) proves this structurally. Not dependent

blocks non-conformant text, regardless of what the model actually

### SC2 — ≥ 90% correct on the 100-question student golden set

**Status: ⚠️ 76% on degraded path (60/79 real responses correct,
21 empty due to quota exhaustion mid-run). Conditionally accepted —
see "Rationale" above.**

- Model used: `llama-3.1-8b-instant` (NOT production's `70b-versatile`)
- Retrieval used: improved keyword-fallback with content-level scoring
  (NOT production's Gemini embedding search)
- Common failure patterns:
  - Model hallucinating facts not in the retrieved chunks (~8 cases)
  - Keyword-match scoring too strict (model gave correct explanation
    but used different phrasing than the expected keyword ~4 cases)
  - Model saying "لا أعرف" when the answer was in the chunk but
    under a different heading/phrasing (~3 cases)
  - Genuinely wrong retrieval (wrong chunk surfaced) ~4 cases
- First run (before retrieval fix): 84/100 but with only 11 real
  responses (89 rate-limited). After improving keyword-fallback with
  content-level scoring: 60/79 real responses passed.

## Retrieval improvement committed in this evaluation pass

`src/nour/knowledge/retriever.py`'s `_keyword_fallback()` was improved
from "grab the first N chunks from matched domains in insertion order"
to a two-stage approach: (1) domain selection via existing
`_KB_CATEGORIES` keywords (unchanged), then (2) content-level scoring
of ALL chunks within matched domains by query-word overlap, returning
the highest-scoring chunks instead of arbitrary first-N. Also: when NO
domain-level keyword matches at all, the improved version searches ALL
allowed domains with content scoring (previously it defaulted to only
`faq.md` + `system_overview.md`, which missed questions about channels,
mobile, troubleshooting, etc.).
