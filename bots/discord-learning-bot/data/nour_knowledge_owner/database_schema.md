# مخطَّط قاعدة البيانات (Database Schema)

> هذا الملف جزء من قاعدة معرفة نور — مخصَّص للمالك فقط. لا يُعرض هذا المحتوى للطلاب أبدًا (Aql Phase A2، domain: `database_schema`).

## نظرة عامة

قاعدة البيانات SQLite واحدة (`data_persist/empire_english.db` داخل حاوية Docker، أو `empire_english.db` محلِّيًّا). كل الجداول مُعرَّفة في `src/database.py` (متغيِّر `_SCHEMA`)، وتُنشَأ تلقائيًّا عند بدء التشغيل عبر `database.init_db()`. الترحيلات الإضافية (إضافة أعمدة جديدة لجداول موجودة) تتم بأمان عبر `_migrate()` — لا يُحذف أو يُعاد تسمية أي عمود أبدًا، فقط إضافات.

## جداول الطلاب والتعلُّم الأساسية

| الجدول | الغرض |
|---|---|
| `members` | السجل الأساسي للطالب: المستوى، المسار، السلسلة، النقاط، الصديق (buddy)، الجنس، الحالة |

| `streaks` | عدد المهام المُنجَزة كل يوم، يُستخدَم لإعادة حساب `current_streak`/`longest_streak` |
| `assessments` | نتائج التقييم الأسبوعي حسب كل بُعد + النتيجة الإجمالية |
| `advancement_exams` | محاولات امتحان الترقية (pending/passed/failed، تسجيلات صوتية، كتابة) |
| `points_log` | سجل تدقيق لكل نقطة مُمنوحة مع السبب |
| `voice_portfolio` | التسجيلات الصوتية المحفوظة (benchmark/daily/assessment) لتتبُّع التقدُّم |
| `vocab_srs` | حالة التكرار المتباعد (SM-2) لمراجعة المفردات لكل طالب/كلمة |
| `ability_milestones` | معالم القدرات المُكتمَلة لكل طالب |
| `conversation_sessions` | جلسات المحادثة الجماعية المُجدوَلة |
| `pronunciation_scores` | نتائج تقييم النطق بالذكاء الاصطناعي (Groq Whisper) |
| `consumed_proof_messages` | رسائل Discord المُستهلَكة كإثبات عمل — تمنع استخدام تسجيل واحد لأكثر من مهمة |
| `done_cooldowns` | آخر وقت `!done` دائم (يبقى بعد إعادة تشغيل البوت) |

## جداول مبادرة Aql (العقل، #15) — نواة نور الذكية

| الجدول | الغرض |
|---|---|
| `knowledge_chunks` | قاعدة المعرفة المُقطَّعة والمُضمَّنة (embeddings) للاسترجاع الذكي (RAG) |
| `nour_episodic_summaries` | ملخَّصات مُكثَّفة لمحادثات سابقة لكل طالب (ذاكرة حِلقية) |
| `journey_coverage` | أعلام بوليَّة مستقلَّة تُحلُّ محلَّ آلة `student_journey` الخطِّية |
| `nour_tool_calls` | سجل مراقبة لكل استدعاء أداة (الاسم، المعطِّيات، الكمون، النجاح/الفشل) |
| `nour_guardrail_events` | سجل مراقبة لحواجز المخرجات (script/bidi/role_leak) |
| `nour_retrieval_log` | سجل مراقبة لجودة الاسترجاع — كل استعلام والمقاطع المُسترجَعة |

**ملاحظة أمنية مهمة:** `knowledge_chunks.domain` هو نفس الحاجز الأمني المُطبَّق في `src/nour/permissions.py`. دالة `database.get_chunks_by_domains(domains)` تُطبِّق شرط `WHERE domain IN (...)` — وهذا الحاجز الثاني (طبقة البيانات) موجود بالإضافة إلى الحاجز الأول (طبقة التطبيق)، دفاعًا في العمق (defense in depth)، لا اعتمادًا على حسن سلوك المُستدعي فقط.

## جداول التقدُّم التكيُّفي والسرد

| الجدول | الغرض |
|---|---|
| ملاحظة | معظم منطق التكيُّف (`adaptive_engine.py`) والسرد (`narrative_engine.py`) يعتمد على الجداول أعلاه (`assessments`, `daily_submissions`, `nour_growth_letters`) دون جداول مخصَّصة إضافية له في الوقت الحالي. |

## الفهارس (Indexes)

كل الجداول عالية الاستخدام لها فهارس مطابقة لأنماط الاستعلام الشائعة (مثل `idx_submissions_date`, `idx_knowledge_domain`, `idx_retrieval_discord`) — راجع `_SCHEMA` في `src/database.py` مباشرة للقائمة الكاملة والدقيقة، فهي تتغيَّر مع كل مبادرة جديدة ولا يُعاد كتابتها هنا يدويًّا لتجنُّب انحراف الوثيقة عن الكود الحقيقي.

## قاعدة ذهبية

هذا الملف وصف بشري لِمَا هو موجود، **وليس** مصدر الحقيقة. مصدر الحقيقة الحقيقي دائمًا هو `_SCHEMA` داخل `src/database.py` — إذا تعارض هذا الملف مع الكود، فالكود هو الصحيح، ويجب تحديث هذا الملف (وإعادة تشغيل سكربت التضمين `scripts/embed_knowledge.py --dry-run` ثم بدونه) لمطابقته.
