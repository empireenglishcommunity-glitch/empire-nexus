# دليل العمليات الإدارية

> هذا الملف جزء من قاعدة معرفة نور — المدربة الذكية في Empire English.

## مقدمة

هذا الملف مخصَّص للمعلومات الإدارية التي تستخدمها نور عند التواصل مع المالك أو المشرفين. لا يُعرض هذا المحتوى للطلاب.

---

## نظام الأعلام (Flags)

### ما هي الأعلام؟
الأعلام هي مفاتيح تشغيل/إيقاف لميزات مختلفة في النظام. تسمح بتفعيل أو تعطيل ميزات بدون تعديل الكود.

### الأوامر:

```
!flag list              — عرض جميع الأعلام وحالتها (مفعَّل/معطَّل)
!flag enable [اسم]     — تفعيل علم محدَّد
!flag disable [اسم]    — تعطيل علم محدَّد
```

### أمثلة على الأعلام:
- `streak_notifications` — إشعارات انقطاع السلسلة
- `daily_reminders` — تذكيرات يومية
- `leaderboard_updates` — تحديثات لوحة المتصدِّرين
- `new_member_welcome` — رسالة ترحيب تلقائية
- `audio_required` — إلزامية إرسال الصوت قبل !done

## أمر المتابعة (!attention)

### الوظيفة:
يعرض قائمة بالأعضاء الذين يحتاجون متابعة خاصة.

### الاستخدام:
```
!attention
```

### يُظهر:
- أعضاء انقطعت سلسلتهم مؤخَّرًا (آخر ٣ أيام).
- أعضاء لم ينشطوا منذ أكثر من أسبوع.
- أعضاء جدد لم يبدأوا بعد.
- أعضاء لديهم أداء منخفض مستمر.

---

## أمر حالة النظام (!status)

### الوظيفة:
عرض الحالة التشغيلية للنظام.

### الاستخدام:
```
!status
```

### يُظهر:
- حالة البوت (متصل/منفصل).
- وقت التشغيل (uptime).
- عدد الأوامر المُعالَجة اليوم.
- حالة قاعدة البيانات.
- حالة منصة التمرين.
- آخر الأخطاء (إن وُجدت).

---

## أمر الأعضاء (!members)

### الوظيفة:
عرض قائمة الأعضاء المسجَّلين مع إحصائياتهم.

### الاستخدام:
```
!members            — قائمة جميع الأعضاء
!members active     — الأعضاء النشطون فقط
!members inactive   — الأعضاء غير النشطين
!members [مستوى]   — أعضاء مستوى محدَّد (مثل: !members l0)
```

### المعلومات المعروضة:
- اسم المستخدم
- المستوى
- تاريخ الانضمام
- طول السلسلة الحالية
- عدد النقاط
- آخر نشاط

---

## إدارة الرموز (!revoke)

### الوظيفة:
إلغاء رمز دخول عضو إلى منصة التمرين.

### الاستخدام:
```
!revoke @username    — إلغاء رمز عضو محدَّد
!revoke all          — إلغاء جميع الرموز (إعادة إصدار تلقائية)
```

### متى يُستخدم:
- عند الاشتباه في مشاركة رمز مع شخص آخر.
- عند حدوث مشكلة أمنية.
- عند حاجة العضو لرمز جديد (يكتب `!link` مرة أخرى).

## نظام البوابة (Role-Gate)

### إعداد البوابة:
```
!setupgate [channel_id] [role_id] [emoji]
```
يُنشئ نظام قبول القوانين الذي يمنح العضو صلاحية الوصول إلى القنوات.

### نشر رسالة البوابة:
```
!postgate [channel_id]
```
ينشر رسالة القوانين مع زر القبول في القناة المحدَّدة.

### كيف يعمل:
1. العضو الجديد يدخل الخادم ولا يرى إلا قنوات محدودة.
2. يذهب إلى `#rules` ويقرأ القوانين.
3. يتفاعل مع الرسالة (يضغط الرمز المحدَّد).
4. يحصل تلقائيًّا على الدور (Role) الذي يفتح باقي القنوات.

## عملية النشر (Deploy)

### الخطوات المعتادة:

```bash
# ١. الاتصال بالخادم
ssh user@server

# ٢. الانتقال إلى مجلد المشروع
cd /path/to/empire-bot

# ٣. سحب آخر التحديثات
git pull origin main

# ٤. إعادة بناء وتشغيل الحاويات
docker compose up -d --build

# ٥. التحقُّق من الحالة
docker compose ps
docker compose logs --tail=50
```

### بعد النشر:
- تحقَّق من حالة البوت في Discord (يجب أن يكون "متصل").
- اكتب `!status` للتأكُّد من عمل جميع الخدمات.
- راقب السجلات لأي أخطاء.

### التراجع عن النشر:
```bash
git log --oneline -5          # معرفة آخر الإصدارات
git checkout [commit_hash]    # العودة لإصدار سابق
docker compose up -d --build  # إعادة البناء
```

## أوامر مركز Telegram

### الأوامر المتاحة:
- `/start` — بدء التفاعل مع بوت Telegram.
- `/status` — حالة النظام.
- `/broadcast [رسالة]` — إرسال رسالة لجميع المشتركين.
- `/stats` — إحصائيات الاستخدام.
- `/sync` — مزامنة البيانات بين Discord وTelegram.

### ملاحظات:
- بوت Telegram هو واجهة إضافية لبعض الوظائف.
- ليس كل الميزات متاحة عبر Telegram.
- البيانات مُشتركة بين النظامين.

## الوصول إلى بيانات الطلاب

### الاستعلامات الشائعة:

```sql
-- عدد الأعضاء النشطين (نشاط في آخر ٧ أيام)
SELECT COUNT(*) FROM members WHERE last_active > NOW() - INTERVAL '7 days';

-- أعلى ١٠ أعضاء بالنقاط
SELECT username, points FROM members ORDER BY points DESC LIMIT 10;

-- أعضاء بسلسلة أكثر من ٣٠ يومًا
SELECT username, streak_days FROM members WHERE streak_days > 30;

-- إحصائيات المهام لهذا الأسبوع
SELECT task_type, COUNT(*) FROM completions 
WHERE completed_at > DATE_TRUNC('week', NOW()) 
GROUP BY task_type;

-- تاريخ عضو محدَّد
SELECT * FROM completions WHERE member_id = [id] ORDER BY completed_at DESC;
```

### ملاحظات أمنية:
- لا تُشارك بيانات الأعضاء الشخصية مع أي طرف.
- الوصول إلى قاعدة البيانات محصور بالمالك والمشرفين المُصرَّح لهم.
- كل استعلام يُسجَّل في سجل العمليات.
